Dead simple parallel execution of async coroutines. n_workers are dispatched which then asynchronously process each task given. Example Usage – EasyAsync(n_workers = 4)(delayed(trio.sleep)(sleep_seconds) for i in range(n_tasks))

Dead easy interface for executing many HTTP requests asynchronously. Check tests for example usage. :Parameters: * n_workers – The number of workers to use. Too many workers will use a lot of memory and increase startup

time, too few can lead to slower execution

  • n_connections – The number of open connections to have

  • progress_bar – ‘notebook’, ‘cli’ or None

  • retries – Number of retries to attempt

  • retry_sleep – How long in seconds to wait when request fails

  • ok_code – A sequence of HTTP status codes to accept as ok

  • check_response_func – A function to apply to the response to determine if ok

  • json – Parse response body as json and return instead of full response

Create a list of requests to pass to EastRequests. Each argument could be a single item or a list. When they are a single item, that attribute is duplicated for every request. All list attributes must have the same length. The list attributes are used as the “describing feature” of each request which aids debugging. Arguments mimic asks.request, which in turn mimics requests.request - see corresponding documentation. :Parameters: * method
